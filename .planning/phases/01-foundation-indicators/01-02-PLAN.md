# Phase 01-02: Leaf/Branch Segment Detection

## Objective
Implement new Leaf/Branch segment detection (YangBranch/YinBranch with N-bar confirmation and back-counting) on top of Leaf. **Only for intervals >= 1h** — sub-hourly noise makes segments unreliable.

## Context
- Reference: Existing `Fish.idr` in glib (kept as Leaf sample)
- New logic: Two-level detection — Leaf for extremum spawning, Branch for N-bar confirmation with back-counting
- Target: Module `Idib.Leaf`
- **Constraint**: Leaf/Branch detection only enabled for Hour1, Hour4, Day1, Week1, Month1, Month3

  constructor MkBranchConfig
  confirmationBars : Nat  -- N = 2 * barsPerMonth(interval), or 0 if disabled
  interval         : Interval

-- Branch only for intervals >= 1h
barsPerMonth : Interval -> Nat
barsPerMonth Month1 = 1
barsPerMonth Month3 = 1
barsPerMonth Week1  = 4
barsPerMonth Day1   = 20
barsPerMonth Hour1  = 130
barsPerMonth Hour4  = 32  -- 130/4
barsPerMonth Min30  = 0   -- DISABLED
barsPerMonth Min15  = 0
barsPerMonth Min5   = 0
barsPerMonth Min1   = 0

defaultBranchConfig : Interval -> BranchConfig
defaultBranchConfig interval = 
  let bp = barsPerMonth interval
  in MkBranchConfig (if bp == 0 then 0 else 2 * bp) interval "SMA7"

-- Only run detection if config.confirmationBars > 0
branchEnabled : BranchConfig -> Bool
branchEnabled config = config.confirmationBars > 0

record LeafBar where
  constructor MkLeafBar
  index : Nat
  value : Double
  chartBar : Maybe (ChartBar i)

  FSSingle : (f : Branch) -> {auto prf : kind f = k} -> BranchSeq k
  FSCons   : (f : Branch) -> {auto prf : kind f = k}
           -> BranchSeq (opposite k) -> BranchSeq k

### Task 4: Branch Detection with N-Bar Confirmation (`src/Idib/Leaf/BranchDetect.idr`)
**Action**: Core algorithm — only runs if `branchEnabled config`

```idris
detectBranch : BranchConfig -> List LeafBar -> List Branch
detectBranch config bars =
  if branchEnabled config
  then ...
  else []  -- Disabled for sub-hourly intervals
  -- 1. Detect all Leaf
  let leaf = detectLeaf bars
  -- 2. Build Branch with N-bar confirmation (N = config.confirmationBars)
  --    YangBranch:
  --    - TRUE start = lowest in subsequent YinLeaf (triggered at new HIGH, back-count)
  --    - Continues until N bars WITHOUT new high
  --    - End = (last_high_index - N)
  --    YinBranch:
  --    - TRUE start = highest in subsequent YangLeaf (triggered at new LOW, back-count)
  --    - Continues until N bars WITHOUT new low
  --    - End = (last_low_index - N)
```

**Algorithm Details (YangBranch):**
1. Find YangLeaf (starts at low)
2. When new HIGH appears (triggering YinLeaf), back-count to LOWEST in that YinLeaf → TRUE Yang start
3. Track "last high index" as new highs appear
4. When N bars pass WITHOUT new high → confirm end at (last_high_index - N)
5. When new LOW appears (ending YinLeaf), back-count to HIGHEST in that YangLeaf → TRUE Yin start
6. Repeat with directions reversed

**Verify**: On test SMA7 at 1h/1d/1wk/1mo, produces reasonable segments; returns [] for 30m/15m/5m/1m
**Done**: Branch detection works with interval gating
