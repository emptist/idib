# Phase 01-02: Fish Segment Detection

## Objective
Implement new Fish segment detection (YangFish/YinFish with N-bar confirmation and back-counting) on top of SimpleFish. **Only for intervals >= 1h** — sub-hourly noise makes segments unreliable.

## Context
- Reference: Existing `Fish.idr` in glib (kept as SimpleFish sample)
- New logic: Two-level detection — SimpleFish for extremum spawning, new Fish for N-bar confirmation with back-counting
- Target: Module `Idib.Fish`
- **Constraint**: Fish detection only enabled for Hour1, Hour4, Day1, Week1, Month1, Month3

### Task 4: New Fish Detection with N-Bar Confirmation (`src/Idib/Fish/Detect.idr`)
**Action**: Core algorithm — only runs if `fishEnabled config`

```idris
detectFish : FishConfig -> List FishBar -> List Fish
detectFish config bars =
  if fishEnabled config
  then ...
  else []  -- Disabled for sub-hourly intervals
  -- 1. Detect all SimpleFish
  let simpleFish = detectSimpleFish bars
  -- 2. Build new Fish with N-bar confirmation (N = config.confirmationBars)
  --    YangFish:
  --    - TRUE start = lowest in subsequent SimpleYin (triggered at new HIGH, back-count)
  --    - Continues until N bars WITHOUT new high
  --    - End = (last_high_index - N)
  --    YinFish:
  --    - TRUE start = highest in subsequent SimpleYang (triggered at new LOW, back-count)
  --    - Continues until N bars WITHOUT new low
  --    - End = (last_low_index - N)
```

**Algorithm Details (YangFish):**
1. Find SimpleYang (starts at low)
2. When new HIGH appears (triggering SimpleYin), back-count to LOWEST in that SimpleYin → TRUE Yang start
3. Track "last high index" as new highs appear
4. When N bars pass WITHOUT new high → confirm end at (last_high_index - N)
5. When new LOW appears (ending SimpleYin), back-count to HIGHEST in that SimpleYang → TRUE Yin start
6. Repeat with directions reversed

**Verify**: On test SMA7 at 1h/1d/1wk/1mo, produces reasonable segments; returns [] for 30m/15m/5m/1m
**Done**: New Fish detection works with interval gating
