# Phase 05-01: Fish Segment Detection

## Objective
Implement Fish segment detection in idib. Two-level detection: SimpleFish (extremum-based) + New Fish (N-bar confirmation + back-counting).

## Context
- Reference: `glib/Fish.idr` (143 lines) — SimpleFish with `So` proofs
- idib README: detailed Fish design with SimpleFish + New Fish + Analytics

## Design (from README)

### Two Fish Types

1. **SimpleFish**: Extremum-based alternating sequence (ported from glib)
   - Yang: starts at lowest value, ends at new HIGH
   - Yin: starts at highest value, ends at new LOW
   - Alternates: Yang → Yin → Yang → ...

2. **New Fish**: Two-level detection with N-bar confirmation
   - N = 2 × barsPerMonth(interval): 1mo=2, 1wk=8, 1d=40, 1h=260, 4h=64
   - YangFish: TRUE start = lowest in subsequent SimpleYin (found at new HIGH)
   - YinFish: TRUE start = highest in subsequent SimpleYang (found at new LOW)
   - **Disabled for sub-hourly** (Min1/5/15/30 return 0 barsPerMonth)

### Key Properties
- **Parameter-free except N** (interval-derived, not tuned) — "natural" segments
- Research record for AI pattern discovery, combined with 3 other techniques later
