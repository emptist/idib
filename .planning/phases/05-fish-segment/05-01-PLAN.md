## Key Insight: Indicators ARE Fish

The existing indicator calculations are **Fish detection applied to SMA7**:

| Indicator | Fish Concept | What It Computes |
|-----------|--------------|------------------|
| `hprd7` | YangFish boundaries | Running count of new peaks in SMA7 |
| `lprd7` | YinFish boundaries | Running count of new lows in SMA7 |
| `hrows7` | Bars within YangFish | Bar index within each peak group |
| `lrows7` | Bars within YinFish | Bar index within each trough group |
| `cmah7` | YangFish mean | Expanding mean within peak groups |
| `cmal7` | YinFish mean | Expanding mean within trough groups |
| `hlcmah7` | Nested Fish mean | Expanding mean within nested fish segments |
| `cnst7` | Fish stability | Expanding mean of (rising && hlhrows == 0) |

The "7" suffix is just the SMA period — the Fish type is **period-agnostic**.

## Fish Type Design

### General Fish (applies to any series)

```idris
-- A fish segment in a time series
record Fish (a : Type) where
  constructor MkFish
  kind      : FishKind           -- Yang or Yin
  startIdx  : Nat                -- start bar index
  endIdx    : Nat                -- end bar index
  startVal  : a                  -- start value
  endVal    : a                  -- end value
  innerBars : List (Nat, a)      -- bars within this fish
  innerFish : List (Fish a)      -- recursive inner fish (fractal)

-- Alternating fish sequence
data FishSeq : FishKind -> Type where
  FSSingle : (f : Fish a) -> {auto prf : kind f = k} -> FishSeq k
  FSCons   : (f : Fish a) -> {auto prf : kind f = k}
          -> FishSeq (opposite k)
          -> FishSeq k
```

### SMA7 Fish (special case)

The existing indicator calculations become:

```idris
-- hprd7 = fish boundaries detected by running peak count
-- lprd7 = fish boundaries detected by running trough count
-- hrows7 = bar index within each hprd7 fish
-- lrows7 = bar index within each lprd7 fish
-- cmah7 = expanding mean within hprd7 fish
-- cmal7 = expanding mean within lprd7 fish
-- hlcmah7 = expanding mean within nested fish

-- These are all computed by Vector.idr functions:
-- runningPeakCount    → hprd (YangFish boundaries)
-- runningTroughCount  → lprd (YinFish boundaries)
-- rowsWithinEachGroup → hrows/lrows (bars within fish)
-- cumulativeMin/MaxWithinGroups → hlsf/hlhsf (extremes within fish)
-- expandingMeanWithinGroups → cmah7/cmal7/hlcmah7 (means within fish)
```

### Recursive Fish Structure

Fish are **fractal** — inside a large Fish, there can be smaller Fish:

```
YangFish (hprd7 group)
  ├── YinFish (hlprd group, nested inside YangFish)
  │     └── YangFish (hlhprd group, nested inside YinFish)
  └── YinFish (another nested segment)
```

This is exactly what the indicator chain computes:
1. `hprd7` → detect primary fish (YangFish boundaries)
2. `hlprd` → detect nested fish within each primary fish
3. `hlhprd` → detect doubly-nested fish within each nested fish

### New Fish with N-Bar Confirmation

For production use (not just SMA7), Fish can have N-bar confirmation:

- N = 2 × barsPerMonth(interval): 1mo=2, 1wk=8, 1d=40, 1h=260, 4h=64
- **Disabled for sub-hourly** (Min1/5/15/30 return 0 barsPerMonth)
- YangFish: TRUE start = lowest in subsequent YinFish (back-count at new HIGH)
- YinFish: TRUE start = highest in subsequent YangFish (back-count at new LOW)

### Key Properties
- **Parameter-free except N** (interval-derived, not tuned) — "natural" segments
- Research record for AI pattern discovery
- Alternation invariant enforced by GADT
- Recursive structure: Fish within Fish (fractal pattern)
