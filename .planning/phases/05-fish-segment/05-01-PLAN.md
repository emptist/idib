# Phase 05-01: Fish Segment Detection

## Objective
Implement new Fish segment detection (YangFish/YinFish with N-bar confirmation and back-counting) on top of SimpleFish. **Only for intervals >= 1h** — sub-hourly noise makes segments unreliable.

## Context
- Reference: Existing `glib/Fish.idr` (143 lines) — SimpleFish with `So` proofs

## Key Insight (from user)

### Naming Convention
- **SimpleFish**: Basic extremum-based alternating sequence (for understanding/concepts)
- **Fish**: Actual implementation with recursive inner Fish and N-bar confirmation

### Recursive Fish Structure

Fish are **fractal** — inside a large Fish, there can be smaller Fish of opposite kind:
- Inside a SimpleYinFish → smaller SimpleYangFish
- Inside a SimpleYangFish → smaller SimpleYinFish
- Same for Fish: inside a large YinFish → smaller YangFish, etc.

This recursive pattern already exists in the indicators:
- `cmal7` (lowest close in window) → tracks low within segment
- `cmah7` (highest close in window) → tracks high within segment  
- `hlcmah7` (highest low of highest closes) → nested tracking
- These calculations are essentially Fish detection without naming it

### New Fish Back-Counting Logic

The "new lowest since current highest" / "new highest since current lowest" describes how to find the **TRUE start** of a Fish:

- **YangFish (rising trend)**:
  - SimpleYang starts at a LOW value, tracks ascent
  - When new HIGH appears (triggering SimpleYin), back-count to find TRUE start
  - TRUE start = lowest value in the subsequent SimpleYin segment
  - This is "new lowest since current highest" — the lowest point since the high that ended the previous segment

- **YinFish (dropping trend)**:
  - SimpleYin starts at a HIGH value, tracks descent
  - When new LOW appears (triggering SimpleYang), back-count to find TRUE start
  - TRUE start = highest value in the subsequent SimpleYang segment
  - This is "new highest since current lowest" — the highest point since the low that ended the previous segment

### SimpleFish Trigger (different from Fish)

SimpleFish uses simpler extremum logic:
- Yang ends when new HIGH exceeds start value
- Yin ends when new LOW falls below start value

### Key Properties
- **Parameter-free except N** (interval-derived, not tuned) — "natural" segments
- Research record for AI pattern discovery
- Alternation invariant enforced by GADT
- Recursive structure: Fish within Fish (fractal pattern)

### New Fish N-Bar Confirmation

- N = 2 × barsPerMonth(interval): 1mo=2, 1wk=8, 1d=40, 1h=260, 4h=64
- **Disabled for sub-hourly** (Min1/5/15/30 return 0 barsPerMonth)
- YangFish: TRUE start = lowest in subsequent SimpleYin (back-count at new HIGH)
- YinFish: TRUE start = highest in subsequent SimpleYang (back-count at new LOW)

### Key Properties
- **Parameter-free except N** (interval-derived, not tuned) — "natural" segments
- Research record for AI pattern discovery
- Alternation invariant enforced by GADT
