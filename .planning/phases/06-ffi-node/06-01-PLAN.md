# Phase 06: Idris 2 → Node FFI

## Objective
Compile idib core to JavaScript, export key functions for Node.js consumption. Enable IB Gateway bridge and SSE server.

## Context
- Idris 2 has built-in Node.js codegen (`--codegen node`)
- Uses `%foreign "node:lambda:..."` for JS interop
- Non-primitive types passed via `AnyPtr`
- glib architecture: Gleam handles logic, JS handles I/O (IB API, HTTP, SSE)

## Architecture

```
┌─────────────────────────────────────────┐
│  Node.js (I/O layer)                    │
│  - IB Gateway connection (@stoqey/ib)   │
│  - HTTP server (dashboard)              │
│  - SSE streaming                        │
│  - setTimeout/setInterval               │
└──────────────┬──────────────────────────┘
               │ FFI calls
               ↓
┌─────────────────────────────────────────┐
│  idib.mjs (compiled from Idris 2)       │
│  - computeChartBars                      │
│  - computeAllChartsFromRaw               │
│  - evaluateSeries                        │
│  - evaluateBarMTF                        │
│  - Leaf/Branch segment detection                │
└─────────────────────────────────────────┘
```

## Tasks

### Task 1: Add FFI exports to Idris modules
**Action**: Create `src/Idib/FFI.idr` with `%foreign` declarations

Key exports needed:
- `computeChartBars` → takes raw bars + config, returns ChartBars
- `evaluateSeries` → takes bars + config, returns signals
- `evaluateBarMTF` → single bar evaluation with multi-TF

### Task 2: Compile to JavaScript
```bash
idris2 --build --codegen node idib.ipkg
```

### Task 3: Create Node.js wrapper
**Action**: Create `ffi/idib-ffi.mjs` that:
- Imports compiled Idris output
- Wraps functions with JSON marshalling
- Exports clean API for IB bridge

### Task 4: Create support file for custom FFI
**Action**: Create `support/js/idib_support.js` with:
- Array marshalling helpers
- Record ↔ object conversion
- Error handling
